# Voynich Lab

Voynich Lab is a documentation-first research repository for reproducible work on the Voynich Manuscript.

The project is not a decipherment claim. Its purpose is to preserve careful observations, experiments, rejected ideas, contradictions, and decisions so that humans, ChatGPT chats, Codex agents, and other models can continue the work without losing context.

## Repository Map

- `main.md` - primary navigation document for new chats and models.
- `philosophy.md` - research philosophy and anti-self-deception rules.
- `project_rules.md` - human-readable operating rules.
- `glossary.md` - recurring terms.
- `knowledge/` - background knowledge, claims, timeline, and bibliography.
- `hypotheses/` - active, planned, weakened, rejected, or superseded hypotheses.
- `experiments/` - experiment concepts, protocols, results, and findings.
- `datasets/` - data sources, provenance, and control corpora.
- `methods/` - transcription, metrics, tests, and visualization rules.
- `logs/` - decisions, session summaries, conflicts, and changelog.
- `prompts/` - templates for Codex and experiment chats.

## How to Start

Start with [main.md](main.md), then read [philosophy.md](philosophy.md), [project_rules.md](project_rules.md), and [glossary.md](glossary.md).

Before running an experiment, check the relevant experiment file in `experiments/`, the current transcription policy in `methods/transcription_policy.md`, and the dataset provenance in `datasets/`.
