# Project Rules

## Delete Nothing Important

Do not delete failed hypotheses, rejected findings, contradictions, outdated notes, or incorrect interpretations. Mark them as `rejected`, `superseded`, `obsolete`, `needs-review`, or `conflict`.

## Separate Observations from Interpretations

Use:

- `OBS` - observation;
- `INF` - interpretation;
- `HYP` - hypothesis;
- `DEC` - decision;
- `TODO` - next step;
- `ERR` - mistake or rejected finding.

## Record Negative Results

Negative results are useful. Preserve them with the same care as positive results.

## Do Not Change Protocols After Results

If a protocol must change, preserve the old version, explain why, and mark the new version clearly.

## Update Documentation After Work

Update experiment files, logs, changelog, and session summaries when work changes the project state.

## Use Sources for Historical and Scientific Information

Do not rely only on memory for manuscript history, dating, materials, ownership, research claims, or bibliography. If a source is missing, mark the entry `needs-source` or `needs-verification`.

## Make No Sensational Claims

Do not claim that the manuscript has been deciphered or that a hypothesis is proven. Use cautious wording and require reproducible evidence.
