# Visualization Rules

## Goals

Charts should make results easier to audit, not more persuasive than the data allows.

## Required Chart Metadata

Each chart should document:

- source data;
- script;
- run date;
- parameters;
- preprocessing;
- limitations.

## Naming

Use stable names such as `token_frequencies.png` or `word_length_distribution.png` under the relevant experiment artifact folder.

## Reproducibility

Charts should be generated by scripts when possible, not manually edited after creation.

## Avoid Misleading Plots

Avoid truncated axes, cherry-picked examples, unlabeled transformations, and visual comparisons without quantitative support.
